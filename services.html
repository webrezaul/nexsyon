Test Automation

For every release a test should be executed to ensure that the software matches the customer requirements. Tests usually take a long time especially when hardware is involved and delays each delivery. Automatizing the tests, in software level, like unit tests is usually fast to implement and to execute. But tests on the target, like integration tests are usually performed manually and require specific external material like oscilloscope, power supply, etc… Tests automation in such context requires control of all test peripherals.
